# 团队RAG智能知识库 — 环境变量配置指南

> 本文档列出了项目运行所需的全部环境变量以及获取方式。
> 请在启动项目前完成所有配置。

---

## 一、环境变量总览

| 环境变量名 | 必需 | 用途 | 示例值 |
|-----------|------|------|--------|
| `DASH_SCOPE_API_KEY` | ✅ 必需 | 阿里百炼平台 API 密钥，用于向量模型和大模型 | `sk-xxxxxxxxxxxxxxxx` |
| `PINECONE_API_KEY` | ✅ 必需 | Pinecone 向量数据库 API 密钥 | `pcsk_xxxxxxxxxxxxxx` |
| `PINECONE_INDEX` | ✅ 必需 | Pinecone 索引名称 | `team-rag-index` |
| `PINECONE_REGION` | ✅ 必需 | Pinecone 服务所在区域 | `us-east-1` |

## 二、详细获取方式

### 2.1 `DASH_SCOPE_API_KEY` — 阿里百炼 API 密钥

此密钥同时用于**向量模型 (text-embedding-v3)** 和**大语言模型 (Qwen-plus)** 的调用。

#### 获取步骤

1. 访问 [阿里百炼平台](https://bailian.console.aliyun.com/)
2. 使用阿里云账号登录（需实名认证）
3. 进入控制台后，点击右上角头像 → 「API-KEY 管理」
4. 点击「创建新的 API-KEY」
5. 复制生成的密钥（格式形如 `sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）

#### 注意事项
- 此密钥包含**免费额度**，新用户通常有一定的免费调用量
- 向量模型 `text-embedding-v3` 的调用费用极低（约 0.0005 元/千 token）
- 大模型 `Qwen-plus` 费用按 token 计费，详见[百炼计费说明](https://help.aliyun.com/document_detail/2712195.html)
- **安全提醒**：不要将密钥提交到 Git 仓库，不要在前端代码中暴露

---

### 2.2 `PINECONE_API_KEY` — Pinecone API 密钥

Pinecone 用于存储和检索文档的向量数据。

#### 获取步骤

1. 访问 [Pinecone 官网](https://www.pinecone.io/)
2. 注册账号（支持 Google / GitHub 登录）
3. 登录后进入 [控制台](https://app.pinecone.io/)
4. 左侧菜单选择「API Keys」
5. 复制 Default 的 API Key（格式形如 `pcsk_xxxxxxxx`）

#### 注意事项
- Pinecone 提供**免费计划** (Starter Plan)，包含：
  - 1 个项目
  - 5 个索引
  - 约 100,000 条向量（1024维）的存储空间
- 免费计划足够团队知识库的初始使用
- 超出额度后需升级付费计划

---

### 2.3 `PINECONE_INDEX` — Pinecone 索引名称

#### 方式一：自动创建（推荐）
项目代码中 `PineconeConfig.java` 已配置自动创建索引功能。只需指定一个索引名即可：
```
PINECONE_INDEX=team-rag-index
```
首次启动时会自动创建该索引，维度为 1024（与 text-embedding-v3 匹配）。

#### 方式二：手动创建
1. 登录 [Pinecone 控制台](https://app.pinecone.io/)
2. 点击「Create Index」
3. 填写信息：
   - **Index Name**: `team-rag-index`（与环境变量保持一致）
   - **Dimensions**: `1024`（⚠️ 必须是 1024，与 text-embedding-v3 一致）
   - **Metric**: `cosine`（余弦相似度）
   - **Cloud**: `AWS`
   - **Region**: 与 `PINECONE_REGION` 保持一致
4. 点击创建

#### 命名建议
- 使用全小写 + 连字符，如 `team-rag-index`
- 不要使用中文或特殊字符
- 不同环境使用不同索引名，如 `team-rag-dev`、`team-rag-prod`

---

### 2.4 `PINECONE_REGION` — Pinecone 区域

根据你在 Pinecone 上选择的云区域设置此变量。

#### 常用区域
| 区域代码 | 位置 | 说明 |
|---------|------|------|
| `us-east-1` | 美国东部（弗吉尼亚） | 免费计划默认区域，推荐 |
| `us-west-2` | 美国西部（俄勒冈） | 备选 |
| `eu-west-1` | 欧洲（爱尔兰） | 适合欧洲用户 |
| `ap-southeast-1` | 亚太（新加坡） | 中国用户延迟较低 |

#### 查看方法
登录 Pinecone 控制台 → 选择你的索引 → 在索引详情页查看 Region 字段。

## 三、配置方式

### 3.1 Windows 本地开发

#### 方式一：系统环境变量（推荐，持久化）

1. 右键「此电脑」→「属性」→「高级系统设置」
2. 点击「环境变量」
3. 在「用户变量」中，逐个添加：

| 变量名 | 变量值 |
|--------|--------|
| `DASH_SCOPE_API_KEY` | 你的阿里百炼密钥 |
| `PINECONE_API_KEY` | 你的Pinecone密钥 |
| `PINECONE_INDEX` | `team-rag-index` |
| `PINECONE_REGION` | `us-east-1` |

4. 点击确定保存
5. **重启 IDE 和终端**使变量生效

#### 方式二：PowerShell 临时设置（仅当前会话有效）
```powershell
$env:DASH_SCOPE_API_KEY = "你的阿里百炼密钥"
$env:PINECONE_API_KEY = "你的Pinecone密钥"
$env:PINECONE_INDEX = "team-rag-index"
$env:PINECONE_REGION = "us-east-1"
```

#### 方式三：IntelliJ IDEA 中配置

1. 打开 Run Configuration（右上角绿色三角旁的下拉 → Edit Configurations）
2. 在 「Environment variables」 栏点击右侧的图标
3. 添加上述 4 个环境变量
4. 点击 OK 保存

### 3.2 Mac / Linux 本地开发

#### 方式一：写入 ~/.bashrc 或 ~/.zshrc（持久化）
```bash
echo 'export DASH_SCOPE_API_KEY="你的阿里百炼密钥"' >> ~/.bashrc
echo 'export PINECONE_API_KEY="你的Pinecone密钥"' >> ~/.bashrc
echo 'export PINECONE_INDEX="team-rag-index"' >> ~/.bashrc
echo 'export PINECONE_REGION="us-east-1"' >> ~/.bashrc
source ~/.bashrc
```

#### 方式二：创建 .env 文件（不提交到 Git）
```bash
# 在项目根目录创建 .env 文件
cat > /opt/team-rag/.env << 'EOF'
DASH_SCOPE_API_KEY=你的阿里百炼密钥
PINECONE_API_KEY=你的Pinecone密钥
PINECONE_INDEX=team-rag-index
PINECONE_REGION=us-east-1
EOF

# 加载环境变量
export $(cat /opt/team-rag/.env | xargs)
```

### 3.3 云端生产环境

参考「云端部署指南.md」中的步骤 4.2，创建 `/opt/team-rag/env.sh` 文件：
```bash
#!/bin/bash
export DASH_SCOPE_API_KEY="你的阿里百炼密钥"
export PINECONE_API_KEY="你的Pinecone密钥"
export PINECONE_INDEX="team-rag-index"
export PINECONE_REGION="us-east-1"
```

在 systemd 服务中通过 `EnvironmentFile` 引用此文件。

## 四、Redis 配置（非环境变量，在配置文件中设置）

Redis 不通过环境变量配置，而是在 `application.properties` 中直接设置：

```properties
# 本地开发（无密码）
spring.data.redis.host=localhost
spring.data.redis.port=6379
spring.data.redis.database=0
spring.data.redis.timeout=5000ms

# 生产环境（需要密码）
# spring.data.redis.password=YourStrongRedisPassword123
```

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `spring.data.redis.host` | `localhost` | Redis 服务器地址 |
| `spring.data.redis.port` | `6379` | Redis 端口 |
| `spring.data.redis.database` | `0` | Redis 数据库编号 |
| `spring.data.redis.password` | 空 | Redis 密码（生产环境必须设置） |
| `spring.data.redis.timeout` | `5000ms` | 连接超时时间 |

## 六、安全提醒

> ⚠️ **以下操作严格禁止：**

1. **不要将 API Key 硬编码在源码中**
   ```java
   // ❌ 错误示范
   .apiKey("sk-xxxxxxxxxxxxxxxx")
   
   // ✅ 正确做法：通过环境变量
   .apiKey(dashScopeApiKey)  // 从 @Value 注入
   ```

2. **不要将 .env 文件提交到 Git 仓库**
   确保 `.gitignore` 中包含：
   ```
   .env
   env.sh
   *.key
   ```

3. **不要在前端代码中使用任何 API Key**
   所有大模型和向量数据库的调用都通过后端完成，前端仅调用后端 API

4. **生产环境定期轮换密钥**
   建议每 3 个月更换一次 API Key，防止泄露风险
