# 团队RAG智能知识库 — 云端部署指南

> 本文档详细描述如何将项目从本地开发环境一步步部署到云端服务器。
> 以**阿里云 ECS**为例说明，腾讯云、华为云等流程类似。

---

## 一、部署架构概览

```
                          ┌──────────────────────┐
                          │       Nginx          │
                          │  (反向代理 + 前端静态) │
                          │  端口: 80 / 443       │
                          └────────┬─────────────┘
                                   │
                    ┌──────────────┼──────────────┐
                    │              │              │
                    ▼              ▼              │
           ┌──────────────┐ ┌──────────┐         │
           │ 前端静态资源   │ │ /api/*   │         │
           │ (React Build) │ │ 反向代理  │         │
           │ /var/www/rag  │ │ → :8080  │         │
           └──────────────┘ └────┬─────┘         │
                                  │               │
                                  ▼               │
                          ┌──────────────┐        │
                          │ Spring Boot  │        │
                          │ 端口: 8080   │        │
                          └──────┬───────┘        │
                                 │                │
                    ┌────────────┼────────┐       │
                    │            │        │       │
                    ▼            ▼        ▼       │
              ┌──────────┐ ┌────────┐ ┌────────┐ │
              │ Redis    │ │Pinecone│ │阿里百炼 │ │
              │ (本地)   │ │ (云端) │ │ (API)  │ │
              │ :6379    │ │        │ │        │ │
              └──────────┘ └────────┘ └────────┘ │
                                                  │
              说明：Pinecone 和 阿里百炼 为 SaaS 服务
              无需自行搭建，通过 API Key 远程调用
```

## 二、购买与准备云服务器

### 2.1 推荐配置

| 配置项 | 最低配置 | 推荐配置 | 说明 |
|--------|---------|---------|------|
| CPU | 2核 | 4核 | Spring Boot + Nginx |
| 内存 | 4GB | 8GB | JVM 需要 2-4GB |
| 磁盘 | 40GB SSD | 80GB SSD | 系统 + 日志 + 上传临时文件 |
| 带宽 | 3Mbps | 5Mbps | 流式输出对带宽要求不高 |
| 系统 | Ubuntu 22.04 | Ubuntu 22.04 LTS | 稳定性好，社区资源丰富 |

### 2.2 购买步骤（以阿里云为例）

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com)
2. 选择「云服务器 ECS」→「创建实例」
3. 选择地域（建议选与 Pinecone 同区域以降低延迟）
4. 选择上述推荐配置
5. 网络选择「按固定带宽」
6. 安全组放行端口：**22**（SSH）、**80**（HTTP）、**443**（HTTPS）、**8080**（测试用，上线后可关闭）
7. 设置 root 密码或 SSH 密钥
8. 确认购买

### 2.3 域名与备案（可选）
- 如需使用域名访问，需提前在阿里云购买域名并完成 ICP 备案
- 备案周期约 7-20 个工作日，建议提前操作
- 未备案前可通过公网 IP 直接访问

## 三、服务器环境安装

通过 SSH 连接到服务器后，依次安装以下组件。

### 3.1 连接服务器
```bash
ssh root@你的服务器公网IP
```

### 3.2 系统更新
```bash
apt update && apt upgrade -y
```

### 3.3 安装 JDK 17
```bash
apt install -y openjdk-17-jdk
java -version
# 输出: openjdk version "17.x.x"
```

### 3.4 安装 Maven
```bash
apt install -y maven
mvn -version
# 输出: Apache Maven 3.x.x
```

### 3.5 安装 Node.js 18+ (用于构建前端)
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs
node -v  # v18.x.x
npm -v   # 9.x.x
```

### 3.6 安装 Redis
```bash
apt install -y redis-server
# 修改配置，设置密码
nano /etc/redis/redis.conf
# 找到 # requirepass foobared，取消注释并修改密码：
# requirepass YourStrongRedisPassword123

# 启动 Redis 并设为开机自启
systemctl start redis-server
systemctl enable redis-server
redis-cli -a YourStrongRedisPassword123 ping
# 输出: PONG
```

> **注意**：生产环境 Redis 必须设置密码，并同步修改后端 `application.properties` 中的 `spring.data.redis.password`。

### 3.7 安装 Nginx
```bash
apt install -y nginx
systemctl start nginx
systemctl enable nginx
# 浏览器访问 http://服务器IP，看到 nginx 默认页面即成功
```

### 3.8 安装 Git
```bash
apt install -y git
```

## 四、部署后端（Spring Boot）

### 4.1 上传代码到服务器

**方式一：Git 拉取**
```bash
mkdir -p /opt/team-rag
cd /opt/team-rag
git clone 你的Git仓库地址 .
```

**方式二：本地打包上传**
```bash
# 本地打包（在 backend 目录执行）
cd e:\chatBox\backend
mvn clean package -DskipTests

# 上传 jar 包到服务器
scp target/team-rag-knowledge-1.0-SNAPSHOT.jar root@服务器IP:/opt/team-rag/
```

### 4.2 配置环境变量
```bash
# 编辑 /etc/profile 或创建专属环境文件
nano /opt/team-rag/env.sh
```

写入以下内容：
```bash
#!/bin/bash
export DASH_SCOPE_API_KEY="你的阿里百炼API密钥"
export PINECONE_API_KEY="你的Pinecone密钥"
export PINECONE_INDEX="你的Pinecone索引名"
export PINECONE_REGION="us-east-1"
```

```bash
chmod +x /opt/team-rag/env.sh
source /opt/team-rag/env.sh
```

### 4.3 生产环境配置文件

创建生产环境配置文件 `application-prod.properties`：
```bash
nano /opt/team-rag/application-prod.properties
```

```properties
# 生产环境覆盖配置
server.port=8080

# Redis（生产环境加密码）
spring.data.redis.host=127.0.0.1
spring.data.redis.port=6379
spring.data.redis.password=YourStrongRedisPassword123

# 日志输出到文件
logging.file.name=/var/log/team-rag/app.log
logging.level.root=info
logging.level.com.team.rag=info
```

### 4.4 创建日志目录
```bash
mkdir -p /var/log/team-rag
```

### 4.5 启动后端服务

**方式一：直接启动 jar 包**
```bash
source /opt/team-rag/env.sh
cd /opt/team-rag

# 如果是 Git 拉取方式，先构建
cd backend && mvn clean package -DskipTests && cd ..

# 启动
nohup java -jar backend/target/team-rag-knowledge-1.0-SNAPSHOT.jar \
  --spring.config.additional-location=/opt/team-rag/application-prod.properties \
  > /var/log/team-rag/startup.log 2>&1 &

# 查看启动日志
tail -f /var/log/team-rag/startup.log
# 看到 "Started TeamRagApplication" 表示启动成功
```

**方式二（推荐）：创建 systemd 服务**
```bash
nano /etc/systemd/system/team-rag.service
```

```ini
[Unit]
Description=Team RAG Knowledge Base
After=network.target redis-server.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/team-rag
EnvironmentFile=/opt/team-rag/env.sh
ExecStart=/usr/bin/java -Xms512m -Xmx2048m \
  -jar /opt/team-rag/backend/target/team-rag-knowledge-1.0-SNAPSHOT.jar \
  --spring.config.additional-location=/opt/team-rag/application-prod.properties
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl start team-rag
systemctl enable team-rag    # 开机自启
systemctl status team-rag    # 查看状态
journalctl -u team-rag -f   # 查看实时日志
```

### 4.6 验证后端
```bash
curl http://localhost:8080/doc.html
# 返回 HTML 内容即表示后端启动成功
```

## 五、部署前端（React 静态资源）

### 5.1 构建生产版本

在服务器上或本地构建：

```bash
cd /opt/team-rag/frontend  # 或本地 e:\chatBox\frontend

# 修改 API 地址为生产环境（不再是 localhost）
```

先修改前端 API 地址，创建生产环境配置：
```bash
nano src/api/chat.ts
# 将 const API_BASE = 'http://localhost:8080/api';
# 改为 const API_BASE = '/api';
# （通过 Nginx 反向代理，前端无需关心后端实际地址）
```

同样修改 `src/api/document.ts` 中的 `API_BASE`。

然后构建：
```bash
npm install
npm run build
# 构建产物在 dist/ 目录
```

### 5.2 部署静态资源
```bash
# 将构建产物复制到 Nginx 目录
mkdir -p /var/www/team-rag
cp -r /opt/team-rag/frontend/dist/* /var/www/team-rag/
```

如果在本地构建，通过 scp 上传：
```bash
scp -r dist/* root@服务器IP:/var/www/team-rag/
```

## 六、配置 Nginx（反向代理 + 静态资源）

### 6.1 创建 Nginx 配置文件
```bash
nano /etc/nginx/sites-available/team-rag
```

```nginx
server {
    listen 80;
    server_name 你的域名或服务器IP;

    # 前端静态资源
    root /var/www/team-rag;
    index index.html;

    # 前端路由（SPA 单页应用）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 流式输出必须配置
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
        proxy_set_header Connection '';
        chunked_transfer_encoding on;
    }

    # Knife4j API 文档（可选，生产环境可关闭）
    location /doc.html {
        proxy_pass http://127.0.0.1:8080/doc.html;
    }
    location /webjars/ {
        proxy_pass http://127.0.0.1:8080/webjars/;
    }
    location /v3/api-docs/ {
        proxy_pass http://127.0.0.1:8080/v3/api-docs/;
    }

    # 上传文件大小限制
    client_max_body_size 50M;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1000;
}
```

### 6.2 启用配置
```bash
# 创建软链接启用站点
ln -s /etc/nginx/sites-available/team-rag /etc/nginx/sites-enabled/

# 移除默认站点（可选）
rm /etc/nginx/sites-enabled/default

# 检查配置语法
nginx -t
# 输出: syntax is ok, test is successful

# 重载 Nginx
systemctl reload nginx
```

### 6.3 验证部署
```bash
# 浏览器访问
http://你的服务器IP

# 或使用域名
http://你的域名
```

## 七、配置 HTTPS（SSL证书）

### 7.1 使用 Let's Encrypt 免费证书（推荐）
```bash
# 安装 Certbot
apt install -y certbot python3-certbot-nginx

# 申请证书（需要域名已解析到服务器IP）
certbot --nginx -d 你的域名

# 按提示操作，Certbot 会自动修改 Nginx 配置添加 SSL
```

### 7.2 使用云平台免费证书

1. 在阿里云/腾讯云申请免费 SSL 证书
2. 下载 Nginx 格式的证书文件
3. 上传到服务器 `/etc/nginx/ssl/` 目录
4. 修改 Nginx 配置启用 HTTPS：

```nginx
server {
    listen 443 ssl;
    server_name 你的域名;

    ssl_certificate /etc/nginx/ssl/your_cert.pem;
    ssl_certificate_key /etc/nginx/ssl/your_key.pem;

    # ... 其余配置同上
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name 你的域名;
    return 301 https://$server_name$request_uri;
}
```

## 十、常见部署问题

### Q1: 后端启动报 "Connection refused" 连接 Redis 失败
- 检查 Redis 是否已启动：`systemctl status redis-server`
- 检查密码是否一致：`application-prod.properties` 中的 `spring.data.redis.password`

### Q2: Nginx 502 Bad Gateway
- 检查后端是否正在运行：`systemctl status team-rag`
- 检查端口是否监听：`ss -tlnp | grep 8080`
- 查看后端日志：`journalctl -u team-rag --since "5 minutes ago"`

### Q3: 前端页面空白
- 检查 `/var/www/team-rag/` 目录下是否有 `index.html` 文件
- 检查 Nginx 配置的 `root` 路径是否正确
- 检查浏览器 F12 Console 是否有 JS 报错

### Q4: SSE 流式输出中断
- 确认 Nginx 配置了 `proxy_buffering off;`
- 确认 `proxy_read_timeout` 设置足够大（建议 300s）
- 检查阿里百炼 API Key 是否有余额

### Q5: 上传大文件失败
- Nginx 端设置：`client_max_body_size 50M;`
- Spring Boot 端设置：`spring.servlet.multipart.max-file-size=50MB`
- 两端都需要配置一致
